# EXP_003 — Grad-CAM Explainability Analysis

## Objective

Investigate which facial regions EfficientNet-B0 uses when classifying images as Real or Fake.

---

## Model

Model: EfficientNet-B0

Checkpoint:
exp002_video_split_baseline.pth

Dataset:
FaceForensics++ Validation Set

Evaluation Protocol:
Video-Level Split

---

## Method

Grad-CAM was applied to randomly selected real and fake validation samples.

Target Layer:
model.features[-1]

The generated heatmaps were overlaid on the original images to visualize model attention.

---

## Observations

Across both real and fake samples, the model consistently focused on:

* Eyes
* Nose
* Mouth
* Central facial region
* Facial texture areas

The model rarely focused on:

* Background regions
* Clothing
* Image borders
* Non-facial regions

---

## Interpretation

The Grad-CAM visualizations suggest that EfficientNet-B0 learned face-specific forensic features rather than relying on dataset-specific shortcuts.

Attention was concentrated on regions commonly affected by deepfake generation:

* Facial blending artifacts
* Mouth synthesis inconsistencies
* Eye-region artifacts
* Texture irregularities

---

## Conclusion

The explainability analysis provides evidence that the detector is making decisions based on meaningful facial information.

This increases confidence that the model is learning genuine deepfake-related features rather than exploiting unrelated visual cues.
