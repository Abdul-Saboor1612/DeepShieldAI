# EXP_002 — Video-Level Split Baseline

## Objective

Evaluate EfficientNet-B0 on FaceForensics++ using a video-level train/validation split to eliminate data leakage between training and validation sets.

---

## Motivation

Experiment 1 used an image-level split where frames from the same video could appear in both training and validation datasets.

This can artificially inflate performance because the model may see highly similar frames during training and evaluation.

Experiment 2 uses a strict video-level split where no video appears in both training and validation sets.

---

## Dataset

Dataset: FaceForensics++ Extracted Frames

Total Images: 191,550

### Training Set

* Total Images: 50,000
* Real Images: 25,000
* Fake Images: 25,000

### Validation Set

* Total Images: 37,439
* Real Images: 6,616
* Fake Images: 30,823

### Leakage Verification

Shared Video IDs Between Train and Validation:

0

## Comparison with EXP_001

EXP_001 (Image-Level Split)

* Validation Accuracy: 96.15%

EXP_002 (Video-Level Split)

* Validation Accuracy: 87.89%

Performance Difference:

-8.26%

---

## Key Findings

1. Image-level splitting significantly overestimates model performance.

2. Video-level splitting provides a more realistic estimate of real-world deepfake detection capability.

3. EfficientNet-B0 remains a strong baseline even after removing train/validation leakage.

4. Future experiments should use video-level splitting as the default evaluation protocol.

---

## Conclusion

The video-level evaluation establishes a trustworthy baseline for DeepFakeGuard.

This experiment will serve as the primary benchmark for future architectures including EfficientNet-B4, Vision Transformers, Grad-CAM explainability, and video-level inference systems.
